객체 지향 프로그래밍이란?
정보와 동작들을 객체 단위로 묶고,이런 객체들을 연결시키면서 정리하는 프로그래밍 방식

A라는 사람과 B라는 사람이 있다 A라는 사람이 은행 계좌에 입금을 하고 출금을 하고, 또 B라는 사람의 계좌에 20,00원을 이체해주려고 한다.
이런 상황을 어떻게 프로그램으로 표현할 수 있을까?

일단 '사람' 이라는 것을 표현할 방법, '계좌'라는 것을 표현할 방법이 필요하다.
그런데 현재까지 배운 자료형들(int,long,arr,boolean)로는 표현하지 어려움.
이런 상황을 쉽게 정리할 수 있는 프로그래밍 방법이 바로 객체 지향 프로그래밍이다.
이 경우에는 '사람'을 표현하는 객체와 '은행 계좌'를 표현하는 객체를 만들 수 있음.
'객체'는 '속성'들과 '동작'들의 묶음이라고 할 수 있는데, 속성은 '변수'로 나타내고, 동작은 '메소드(함수)'로 나타냄.

## Class
### 사람
'클래스'는 앞서 말한 객체들에 대한 설계도, 설명서임. '사람'객체에 대한 설명서를 먼저 써보면 알게됨

```java
public class Person {
    String name;
    int age;
    int cashAmount;
    // 은행 계좌?
}
```
Person이라는 클래스를 만들고 '이름'을 표현하는 name이라는 문자열 변수,'나이'를 표현하는 age라는 int형 변수, '보유 현금'을 표현하는 cashAmount라는 int형 변수가 있음.
'은행계좌'를 표현하는 변수는 비워두고

### 은행계좌
'은행계좌' 객체에 대한 설명서를 써보면 
```java
public class BankAccount {
    int balance;
    // 소유인?
}
```
클래스를 만들고, 이 클래스에는 '잔액'을 표현하는 balance라는 int형 변수가 있음.
'소유인'을 표현하는 변수는 마찬가지로 비워둠

### 연결
이제 비워뒀던 '소유인' 과 '은행계좌'의 관계를 보면
```java
public class Person {
    String name;
    int age;
    int cashAmount;
    BankAccount account;
}
```
BankAccount 클래스를 자료형처럼 사용한걸 볼 수 있는데, account 변수는 BankAccount 타입을 갖게 됨
```java
public class BankAccount {
    int balance;
    Person owner;
}
```
마찬가지로 owner 변수는 Person타입을 가지게 됨.
이런 식으로 프로그램을 객체 단위로 정리하고, 객체들끼리 엮으면서 프로그래밍하는 방식이 바로 '객체 지향 프로그래밍'이다.
